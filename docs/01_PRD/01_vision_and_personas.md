# Module 1: Product Requirement Document (PRD)

## 1.1 — Product Vision

**The Remainder Portal** is a bespoke, invitation-only community platform for a private aristocratic roleplay collective. It exists to replace the brittle, algorithmically hostile environment of Facebook with a sovereign, self-governed digital estate — one that reflects the elegance of its lore. The application is not merely a utility; it is an *experience artefact*. Every pixel, interaction spring, and AI-graded narrative thread must reinforce the community's worldbuilding identity.

### Core Value Propositions

| Value | Description |
|---|---|
| **Sovereignty** | Community owns its data, pipelines, and moderation logic |
| **Elegance** | Design language is as deliberate as the lore itself |
| **Intelligence** | Multi-model AI grades applications, summarizes plots, and guards canon |
| **Mobility** | Every view is mobile-first and degrades gracefully up to 4K desktop |
| **Continuity** | Narrative state is permanent, RAG-backed, and hallucination-resistant |

## 1.2 — User Personas

### Persona A: The Player (`role: player`)

> *"I want to explore the world, track my character, read the Chronicles, and feel like I belong to something real."*

- **Device:** Primarily iPhone or Android flagship, occasionally desktop
- **Behavior:** Passive-to-moderate: reads Chronicle timelines, checks roster, manages character card
- **Pain Points (Current Facebook):** Algorithm buries important posts, no searchable roster, narrative threads get lost
- **Goals:** Beautiful immersive experience, instant information access, no friction

### Persona B: The Administrator (`role: admin`)

> *"I need to process applications accurately, manage faceclaims, update roster records, and ensure lore integrity — ideally from my phone during downtime."*

- **Device:** Mix of mobile and desktop; must handle admin tasks from mobile on-the-go
- **Behavior:** Active: reviews applications, approves/declines members, updates Google Sheets, reads AI evaluation summaries
- **Pain Points (Current):** Manual evaluation, cross-referencing lore docs, no structured pipeline
- **Goals:** Fast mobile-optimized pipeline, AI-assisted grading, one-tap approval workflow
